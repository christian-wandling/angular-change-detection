Scenario
- A parent component containing a User object with 3 child components displaying the name property of the User object.
- For the first two child components the parent passes the User object to the children via the Input interface.
- The first child component uses the default change detection, the second uses OnPush.
- The third child component uses the same service as the parent and uses a subscription to listen for changes in the User object.
Solution

By using the async pipe and its in-built change detection instead of a subscription we make sure this child component is updated as well.
